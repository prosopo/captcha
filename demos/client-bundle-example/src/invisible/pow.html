Procaptcha Invisible Mode - Proof of Work

This example demonstrates how to use Procaptcha with Proof of Work verification in invisible mode.

Silent Verification: Unlike regular CAPTCHA, the Proof of Work verification happens completely in the background without displaying any modal or UI to the user. The process is:
  1. User fills out the form and clicks Submit
  2. The execute() function is called, triggering the custom event
  3. PoW component silently performs the verification computation
  4. Upon success, the callback is called with the verification token
  5. No user interaction is required during verification

How It Works

This example demonstrates how to use Procaptcha with Proof of Work verification in invisible mode:

  1. Include the Procaptcha script with ?render=explicit parameter
  2. Create a container div with data-size="invisible"
  3. Specify 'captchaType': 'pow' when rendering the widget
  4. When the submit button is clicked, call window.procaptcha.execute()
  5. The PoW verification happens silently in the background
  6. When verification completes, your callback function is called with the token

Event-Based Communication System

This implementation uses a custom event system to communicate between components:

  1. When window.procaptcha.execute() is called, it dispatches a procaptcha:execute event
  2. The PoW ProcaptchaWidget component listens for this event
  3. When the event is received, the component silently initiates verification
  4. No modal or UI is displayed during the process
  5. After verification, the callback is called with the result

This approach allows for a completely frictionless user experience with background verification.
