How Frictionless CAPTCHA Works (Explicit Rendering)

Implementation Details

This demo shows how to implement frictionless CAPTCHA using explicit rendering:

  1. Import the Procaptcha render function
  2. Create a container div where the CAPTCHA will be rendered
  3. Call the render function with the container and configuration options
  4. Specify callback functions for verification and failure

Key Code Example

import { render } from "%VITE_BUNDLE_URL%"
import { CaptchaType } from "@prosopo/types";

function onCaptchaVerified(output) {
    console.log('Challenge passed')
}

const captchaContainer = document.getElementById('procaptcha-container')
const widgetId = render(captchaContainer, {
    siteKey: "%PROSOPO_SITE_KEY_FRICTIONLESS%",
    captchaType: CaptchaType.frictionless,
    callback: onCaptchaVerified,
    "failed-callback": function () {
        console.log('Challenge failed')
    }
});

Execution Flow

  1. On page load, the render function is called to initialize the CAPTCHA
  2. The CAPTCHA verification happens invisibly in the background
  3. When verification is complete, the callback function is called with the token
  4. The token is stored in a hidden input with name='procaptcha-response'
  5. When the form is submitted, the token is included in the request
