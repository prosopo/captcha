How Invisible Image CAPTCHA Works (Explicit Rendering)

Implementation Details

This demo shows how to implement invisible Image CAPTCHA with explicit rendering:

  1. Include the Procaptcha script
  2. Create a container div with data-size="invisible"
  3. Define an onloadCallback function that renders the Procaptcha widget
  4. Add a click handler to your submit button that validates the form and executes Procaptcha
  5. When Procaptcha verification succeeds, your callback function is called

Key Elements

<div id="procaptcha-container" data-size="invisible"></div>

<script>
// Render the widget
const widgetId = render(document.getElementById('procaptcha-container'), {
    siteKey: "%PROSOPO_SITE_KEY_IMAGE%", 
    captchaType: "image",
    callback: onSubmit,
    size: "invisible"
});

// Execute the verification when needed
window.procaptcha.execute();
</script>

Execution Flow

  1. The script loads and renders the CAPTCHA into the specified container
  2. When the submit button is clicked, the form data is validated
  3. If validation passes, procaptcha.execute() is called
  4. This dispatches a procaptcha:execute event
  5. The CAPTCHA verification happens invisibly in the background
  6. When verification completes, your callback receives the token
