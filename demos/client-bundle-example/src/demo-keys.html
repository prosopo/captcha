What are Demo Keys?

Demo keys are special hardcoded site keys that provide predictable CAPTCHA behavior for testing and development purposes. They allow developers to test both success and failure scenarios without requiring a real provider setup.

Why Use Demo Keys?

  • Testing: Verify your integration handles both success and failure cases
  • Development: Build and debug your application without provider dependencies
  • CI/CD: Run automated tests with predictable CAPTCHA responses
  • Demos: Show CAPTCHA functionality in controlled environments

üîß Implementation Notes

Frontend (Widget) Behavior

  • Demo keys are detected by comparing the site key to hardcoded demo key values
  • A prominent red warning banner is displayed at the bottom of the widget
  • Banner shows the demo mode type: "DEMO MODE (ALWAYS PASS)" or "DEMO MODE (ALWAYS FAIL)"
  • Banner includes a "Learn More" link to documentation

Backend (Provider) Behavior

  • Always Pass: Returns trivial PoW challenges (difficulty 1) and always verifies as successful
  • Always Fail: Returns the same trivial challenges but always fails verification
  • All demo key usage is logged with warnings in provider logs
  • Demo keys are automatically disabled in production environments

Use Cases

  • Unit Tests: Test success/failure handling without mocking
  • Integration Tests: Verify full flow with predictable outcomes
  • UI Development: Build and style success/error states
  • Demos & Presentations: Show functionality without complexity
